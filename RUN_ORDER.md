# Run Order

This document describes the recommended execution order for the Code Ocean capsule and for local reproducibility.

The repository is organized for Code Ocean as follows:

```text
/code/      analysis scripts and runner files
/data/      input data mounted by Code Ocean
/results/   exported results generated by Code Ocean
```

The recommended Code Ocean run command is:

```bash
Rscript run_codeocean.R
```

`run_codeocean.R` is the stable entry point. It prepares input paths, runs available analysis steps in dependency order, records skipped or failed steps, and copies generated output files to `/results` without renaming or changing result formats.

## 1. Input data

Upload the GEO supplementary archive to the Code Ocean Data section as:

```text
/data/GSE138709_RAW.tar
```

The runner also recognizes extracted UMI matrices in:

```text
/data/GSE138709_RAW/
```

Processed Seurat objects can also be supplied if available:

```text
/data/scRNA1_preprocessed.rds
/data/output/scRNA1_preprocessed.rds
/data/scRNA1_annotated.rds
/data/output/scRNA1_annotated.rds
```

## 2. Environment setup

In a clean Code Ocean capsule, run the following commands from `/code`:

```bash
bash codeocean_system_deps.sh
Rscript 00_install_core_packages.R
Rscript packages.R
Rscript run_codeocean.R
```

If system libraries are already available, use:

```bash
Rscript 00_install_core_packages.R
Rscript packages.R
Rscript run_codeocean.R
```

For routine Code Ocean execution after setup, use:

```bash
Rscript run_codeocean.R
```

## 3. Default reviewer-safe run

By default, `run_codeocean.R` runs the core steps and skips heavy optional steps when their packages or input objects are unavailable.

Core steps include preprocessing, annotation, sample-origin visualization, and YBX1 expression projection. Heavy optional steps include CNV inference, Monocle trajectory analysis, drug-sensitivity prediction, SCENIC, GSVA, regulon heatmaps, and density plots requiring heavy intermediate objects.

Skipped steps are recorded in:

```text
output/codeocean_run_log.csv
/results/output/codeocean_run_log.csv
```

## 4. Full run with heavy steps

After installing all optional packages and preparing the required intermediate objects/databases, the full workflow can be run with:

```bash
RUN_HEAVY_STEPS=true STRICT_CORE=true Rscript run_codeocean.R
```

Heavy optional packages include:

```text
monocle3
infercnv
copykat
SCENIC
oncoPredict
AUCell
RcisTarget
GENIE3
GSVA
ComplexHeatmap
Nebulosa
```

SCENIC-related steps also require the corresponding motif-ranking database files.

The root-level ordered scripts are standalone scripts. They do not require `source("legacy_scripts/...")`.

Although the file names are numbered from 01 to 18, the execution order used by `run_codeocean.R` is dependency-aware. For example, the box-density script is run after CopyKAT, drug-sensitivity prediction, SCENIC AUC integration, and GSVA because it uses those upstream metadata columns and scores.

## 7. Output locations

Original script outputs are preserved. Most files are written to:

```text
/code/output/
```

At the end of `run_codeocean.R`, generated results are copied to:

```text
/results/output/
/results/
```

The copy step does not rename files and does not modify PDF, PNG, CSV, TSV, TXT, or RDS formats.

A manifest of exported files is written to:

```text
/results/result_manifest.csv
```

## 8. Backup scripts

The `legacy_scripts/` directory contains archived copies of the original long-name implementation scripts. It is kept for traceability only. Routine Code Ocean runs should use the standalone root-level scripts listed above.
